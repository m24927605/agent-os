# SLICE-P0-001: 在 ids 與 AuditEvent 上完成 OCSF AgentContext 對映

- **Phase**: P0（對應 architecture-approach.md §4「Phase 0（延續 scaffold，非重寫）」）
- **Branch**: slice/p0-001-ocsf-agentcontext
- **Author**: <id>    **Adversarial reviewer**: <須為 fresh-context、非作者>
- **Size budget**: 估計 <= 0.5 day；預期 net LOC <~120、files <~5、modules <~2（`src/iam` + `src/audit`）

## (1) ID + Title
SLICE-P0-001 — 把 scaffold 既有的 branded ids 聚合成一個**型別化、runtime-validated 的 OCSF AgentContext**，並讓 `createAuditEvent` 以該 context 為單一來源拼裝事件（消除欄位漂移）。

## (2) Goal（一句話）
新增一個 `AgentContext`（actorId/tenantId/projectId/taskId/requestId/sandboxId?）的 Zod schema + 建構函式，使每個 AuditEvent 的身分欄位**只能**經此 context 取得、且缺欄位 fail-closed throw。

## (3) In-scope / Out-of-scope
- In-scope:
  - 在 `src/iam/` 新增 `AgentContext` Zod schema（聚合既有 7 個 branded ids；`sandboxId` optional）+ `parseAgentContext(input: unknown): AgentContext`（fail-closed）。
  - `createAuditEvent` 接受一個 `agentContext`（或從既有 input 衍生）並由它填身分欄位；身分欄位與 action/resource/policyDecision/result 解耦。
  - barrel `src/index.ts` 導出 `AgentContext` 公共面。
- Out-of-scope（明確不做）:
  - OCSF v1.7.0 wire-shape 序列化對映 → 留給 **SLICE-P0-002**（canonical serialize）與 P1 evidence kernel ingest。
  - 把 PolicyRequest 也改用 AgentContext → **非 Phase-0 依賴**；留給一個**可選的後續 slice（SLICE-P0-007）**。（注意：SLICE-P0-006 PDP seed **刻意不**消費 AgentContext、只依賴 S0.3，見其 §3。）
  - 任何跨租 runtime 隔離強制 → 留給 **P3**（gateway-per-tenant）。

## (4) Design delta + modules + public interface + dependency direction
- **Design delta**: 目前 `AuditEvent` 的身分欄位（tenantId/projectId/...）散落在 `AuditEventInput` 的扁平欄位。新增 `AgentContext` 將其聚合為一個 domain value object；`createAuditEvent` 改為「context + action 細節」兩段，行為相容（既有測試不破）但身分來源唯一化。**不改** AuditEvent 對外 JSON 形狀。
- **Modules touched（每個一句唯一責任，high cohesion 自證）**:
  - `src/iam/ids.ts` — **唯一責任**：定義身分 primitives（branded ids + 其聚合 AgentContext 契約與 fail-closed 解析）。
  - `src/audit/event.ts` — **唯一責任**：把一個已驗證的 AgentContext + action 細節組成完整、不可 partial 的 AuditEvent。
- **PUBLIC interface（新增/變更的對外公共面；內部實作不列）**:
  - `export const AgentContext: z.ZodType<…>`（actorId/tenantId/projectId/taskId/requestId/sandboxId?）
  - `export type AgentContext = z.infer<typeof AgentContext>`
  - `export function parseAgentContext(input: unknown): AgentContext`（缺欄位/空字串 → throw，fail-closed）
  - `createAuditEvent` 簽章保持向後相容：`AuditEventInput` 既有扁平欄位仍可用；新增 overload/內部以 `parseAgentContext` 收斂身分欄位。
- **Dependency direction（low coupling 自證；HARD CONSTRAINT A）**:
  - 箭頭圖（向內、無 cycle）:
    ```
    src/audit (application/domain) ──▶ src/iam (domain primitives)
    ```
    （audit 依賴 iam；iam 不依賴 audit；維持既有方向、無新反向邊、無 cycle。）
  - 僅經 public surface 消費（無 deep import）: ☑ 是（audit 只 import `../iam/ids.js`，不碰 iam 內部其他檔）。
  - 新依賴宣告: **無新第三方依賴**（沿用既有 `zod`）。無新跨 module 依賴（audit→iam 既已存在）。

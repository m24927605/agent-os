## (1) ID + Title
SLICE-P0-004 — 定義 OpenShell adapter 的**最小 TS interface**（sandbox lifecycle：create/start/stop/destroy 的型別化契約）+ 一個**fail-closed null adapter**（所有操作預設 deny / 不執行任何外部副作用）+ **mock contract tests**（無需運行中的 OpenShell）。

## (2) Goal（一句話）
新增 `SandboxAdapter` interface 與 `NullSandboxAdapter`（每個方法回 deny-by-default 的結構化結果、絕不觸發外部副作用），並以 mock contract test 鎖定「未配置的 adapter 一律拒絕且可被審計」。

## (3) In-scope / Out-of-scope
- In-scope:
  - `src/runtime/openshell/adapter.ts`：`SandboxAdapter` interface（方法：`createSandbox` / `startSandbox` / `stopSandbox` / `destroySandbox`；每個收 AgentContext + 最小參數，回 `AdapterResult`（discriminated union：`ok` | `denied`）。
  - `NullSandboxAdapter`：所有方法回 `{ status: "denied", reason: "no OpenShell adapter configured (deny-by-default, fail-closed)" }`，**不做任何 I/O / network / process**。
  - mock contract test：以 in-memory mock 驗證 interface 形狀 + null adapter 的 deny 行為 + 每次操作可產生 sandbox-lifecycle 事件骨架（create/start/stop/destroy 為可審計事件，呼應 CLAUDE.md「Sandbox lifecycle events are auditable」）。
- Out-of-scope（明確不做）:
  - 真實 `connect-node` gRPC client / `proto/openshell.proto` 綁定 / pin image digest → 留給 **P2**（Personal beachhead 串本機 OpenShell）。
  - Exec/credential resolve/inference RPC → 留給 P2/P3。
  - 把 lifecycle 事件真正寫入 evidence kernel → 留給 **SLICE-P0-005**（v0 契約）+ P1（真實 ingest）；本 slice 只回**事件骨架**，不 append。

## (4) Design delta + modules + public interface + dependency direction
- **Design delta**: 目前無 runtime adapter（`src/index.ts` 註明「later task under src/runtime/openshell」）。本 slice 立**契約 + 安全預設值**：在沒有任何真實 backend 時，adapter 必須**結構性 deny**（managed-path-is-only-path：未配置 = 不放行）。這把「adapter 是 single chokepoint」從口號變成可被測試的 fail-closed 預設。
- **Modules touched（每個一句唯一責任，high cohesion 自證）**:
  - `src/runtime/openshell/adapter.ts`（新增）— **唯一責任**：定義 sandbox lifecycle 的型別化契約與其 deny-by-default 預設實作；**不含**任何 policy 決策邏輯（policy 在 PDP，不得洩進 adapter）。
- **PUBLIC interface（新增/變更的對外公共面）**:
  - `export interface SandboxAdapter { createSandbox(ctx, spec): Promise<AdapterResult>; startSandbox(...); stopSandbox(...); destroySandbox(...); }`
  - `export type AdapterResult = { status: "ok"; sandboxId: SandboxId; event: SandboxLifecycleEvent } | { status: "denied"; reason: string; event: SandboxLifecycleEvent }`
  - `export class NullSandboxAdapter implements SandboxAdapter`（所有方法 deny）
  - `export const SandboxLifecycleEvent`（Zod；最小骨架，攜 AgentContext + `lifecycle: "create"|"start"|"stop"|"destroy"` + result）
- **Dependency direction（low coupling 自證；HARD CONSTRAINT A）**:
  - 箭頭圖（向內、無 cycle）:
    ```
    src/runtime/openshell (adapter) ──▶ src/iam (AgentContext, SandboxId)
    ```
    （adapter 是 outer layer，依賴向內指向 domain primitives；**不**反向、**不** import policy/audit 內部。）
  - 僅經 public surface 消費（無 deep import）: ☑ 是（只 import `../../iam/ids.js`）。
  - 新依賴宣告: **無新第三方依賴**（無 connect-node；本 slice 純 interface + null impl）。**禁止** adapter import policy 內部以「順手做決策」——policy 決策屬 PDP（S0.6），洩進 adapter 即 §4.7 BLOCKER。

## (5) Test-first plan（先寫的 RED 測試）
- 測試檔: `src/runtime/openshell/adapter.test.ts`（新增）
- RED 測試清單（每條對應一個行為/不變量）:
  - [ ] **Deny-by-default**：`NullSandboxAdapter.createSandbox(ctx, spec)` → `status: "denied"`，reason 含 `deny-by-default`。
  - [ ] **Fail-closed / no side effect（結構性，非僅 mock 兩個 API）**：null adapter **結構上不可能**有 egress——
    斷言方式**首選結構性**：`adapter.ts` 的 import 圖**不含任何 I/O-capable module**（`node:http`/`node:https`/
    `node:net`/`node:dns`/`node:child_process`/`node:fs`/`node:worker_threads`/`fetch`/動態 `import()`），由
    `pnpm run deps:check` 的規則或一條「import allowlist」測試守住（只允許 `../../iam/ids.js` + `zod`）。
    mock 監看 fetch/child_process 為 0 次呼叫**僅作輔助**，不可作為唯一證據（Node egress 路徑眾多，純 mock 兩個
    具名 API 可能漏掉 `node:net`/`node:dns`/dynamic import 等真實逃逸路徑）。
  - [ ] **Lifecycle auditability**：每個方法回傳的 `event` 為合法 `SandboxLifecycleEvent`（攜完整 AgentContext + lifecycle + result=denied）。
  - [ ] **No policy leakage（cohesion）**：adapter 不接受、不評估任何 allow rule（型別層即無 rule 參數）——以「adapter 介面不含 policy 概念」的測試/型別斷言守。
  - [ ] 安全對抗式（fail-closed）: 傳入殘缺 AgentContext → adapter 仍 deny（不 throw 成被上層當成功，也不 allow）。

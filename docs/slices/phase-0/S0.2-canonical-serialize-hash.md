# SLICE-P0-002: AuditEvent canonical serialization + content-addressed hashing

## (1) ID + Title
SLICE-P0-002 — 為 AuditEvent 加上**唯一、deterministic 的 canonical byte serialization**，並由其位元組產生**content-addressed hash**（content id），使同一事件無論欄位順序皆得相同 hash——這是 P1 hash-chain / WORM 的前置不變量。

## (2) Goal（一句話）
新增 `canonicalizeAuditEvent(event): Uint8Array`（rejects 非有限數、確定鍵序、redaction-after）與 `contentAddress(event): string`（對 canonical bytes 取 SHA-256），保證 byte-for-byte 與 hash 的可重現性。

## (3) In-scope / Out-of-scope
- In-scope:
  - `src/audit/canonical.ts`：把 AuditEvent 轉成**確定性**位元組序列（遞迴 key 排序、UTF-8、拒絕 `NaN`/`Infinity`/`undefined`、無浮點漂移風險欄位）。
  - `contentAddress(event): string`（`sha256:<hex>`）——content-addressed id，供 P1 hash-chain 的 leaf。
  - serialization **必須先 redact 再 canonicalize**（沿用 `redactSecrets`），維持 6-sink credential 不變量。
  - barrel 導出公共面。
- Out-of-scope（明確不做）:
  - 實際 hash-chain（prev_hash 串接）→ 留給 **SLICE-P0-005**（kernel v0 契約）。
  - Ed25519 簽章 / RFC-3161 外部錨定 / Tessera tile → 留給 **P1**。
  - BLAKE3（architecture-approach.md 目標）→ Phase 0 先用 Node 內建 `node:crypto` SHA-256（零新依賴）；P1 kernel 改 BLAKE3，content-address 介面**版本化**（見 §4 設計）以容後替換。

## (4) Design delta + modules + public interface + dependency direction
- **Design delta**: 目前 `serializeAuditEvent` = `JSON.stringify(redactSecrets(event))`——`JSON.stringify` 的鍵序**不保證確定**且接受 `NaN`→`null` 之類靜默轉換。新增一條**確定性**路徑：固定鍵序（遞迴字典序）、拒絕非有限/`undefined`、明確 hash 演算法前綴（`sha256:`），使 hash 可被獨立 verifier 重算。`serializeAuditEvent`（人類/log 用）保留；canonical 路徑（機器/chain 用）並存、互不取代。
- **Modules touched（每個一句唯一責任，high cohesion 自證）**:
  - `src/audit/canonical.ts`（新增）— **唯一責任**：把 AuditEvent 轉為唯一 canonical bytes 並計其 content address。
- **PUBLIC interface（新增/變更的對外公共面）**:
  - `export function canonicalizeAuditEvent(event: AuditEvent): Uint8Array`
  - `export function contentAddress(event: AuditEvent): string`  // 回傳 `"sha256:<hex>"`，前綴使演算法可演進（P1→blake3:）
  - （不變更 `serializeAuditEvent` 既有簽章）
- **Dependency direction（low coupling 自證；HARD CONSTRAINT A）**:
  - 箭頭圖（向內、無 cycle）:
    ```
    src/audit/canonical ──▶ src/audit/event (AuditEvent type)
                        ──▶ src/audit/redact (redactSecrets)
                        ──▶ node:crypto (平台原生)
    ```
    （皆在 audit module 內 + 平台 stdlib；無跨 module 新邊、無 cycle、向內。）
  - 僅經 public surface 消費（無 deep import）: ☑ 是（同 module 內檔案間 import 屬內聚實作；對外只導出兩個函式）。
  - 新依賴宣告: **無新第三方依賴**（`node:crypto` 為平台原生）。

## (5) Test-first plan（先寫的 RED 測試）
- 測試檔: `src/audit/canonical.test.ts`（新增）
- RED 測試清單（每條對應一個行為/不變量）:
  - [ ] **Determinism**：同一 event、以不同物件鍵插入順序建構 → `canonicalizeAuditEvent` 產生 **byte-identical** 輸出；`contentAddress` 相同。
  - [ ] **Distinctness**：改任一欄位（例如 `result: "denied"→"success"`）→ content address **不同**（hash 對內容敏感）。
  - [ ] **Format**：`contentAddress` 回傳 `^sha256:[0-9a-f]{64}$`。
  - [ ] 安全對抗式（fail-closed）: 含非有限數 / `undefined` 巢狀值 → canonicalize **throw**（不靜默轉 `null`），避免 hash 二義性。
  - [ ] 安全對抗式（credential non-leak）: event.context 帶 `apiKey` → canonical bytes **不含**該值（先 redact 後 canonicalize；以 canary 字串 assert 0 命中）。

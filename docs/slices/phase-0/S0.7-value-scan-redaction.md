# SLICE-P0-007: enforced value-scanning redaction filter（補 by-key redaction 的缺口）

- **Phase**: P0（INDEX §0「尚待補的 Phase 0 slice」；engineering-standards §7.3、test-and-acceptance §3.2）
- **Branch**: slice/p0-007-value-scan-redaction
- **Author**: <id>    **Adversarial reviewer**: <須為 fresh-context、非作者>
- **Size budget**: 估計 <= 0.5 day；預期 net LOC <~90、files <~2、modules <~1（`src/audit`）

## (1) ID + Title
SLICE-P0-007 — 在既有 by-key `redactSecrets` 之上加 **value-scanning**：對任何字串值（即使在非 secret-named 欄位）掃描高訊號 secret 形狀並 scrub，使 secret 走漏進 free-form 欄位也不會進入 canonical bytes / hash chain / audit 輸出。

## (2) Goal（一句話）
讓 `redactSecrets` 對字串值掃描已知 secret pattern（`sk-…`/`gh[pousr]_…`/`AKIA…`/`xox…`/PEM private key/JWT）並以 `[REDACTED]` 取代命中子字串，關閉「secret 在非 secret key 下存活」的洩漏路徑。

## (3) In-scope / Out-of-scope
- In-scope:
  - `src/audit/redact.ts`：在遞迴 redaction 中對 string 值套用 value-scan（保留既有 by-key 行為）。
  - 高訊號 pattern（與 `scripts/scan_secrets.sh` 對齊，低誤判）；命中即以 `[REDACTED]` 取代命中片段。
  - 因 `canonicalizeAuditEvent` 已先 redact 再 hash（S0.2），此 scrub 自動覆蓋 entryHash / chain（S0.5）/ serialize 出口。
- Out-of-scope（明確不做）:
  - 高熵通用偵測 / entropy scoring（誤判風險高）→ 後續可選。
  - 把 scrub 擴到 `scripts/scan_secrets.sh` 的 6-sink 掃描覆蓋（logs/artifacts/snapshots/traces）→ 屬 scanner 的後續強化。
  - 任何 PEP/policy 變更。

## (4) Design delta + modules + public interface + dependency direction
- **Design delta**: 目前 `redact()` 只在 KEY 命中 secret-name 時整值 `[REDACTED]`；string 值原樣通過。新增：string 值經 `scrubSecretValues` 掃描並取代命中片段。by-key 行為不變（secret-named key 仍整值 redact）。
- **Modules touched**:
  - `src/audit/redact.ts` — **唯一責任**：把任何結構中的 secret-like 內容（by-key + by-value）換成 `[REDACTED]`。
- **PUBLIC interface（不新增對外簽章）**:
  - `redactSecrets<T>(value: T): T` 簽章不變（行為加強）。可選 export `REDACTED`（已有）。
- **Dependency direction（low coupling）**:
  - redact.ts 無 import（純函式，零依賴）；不新增任何邊。`deps:check` 不受影響。

## (5) Test-first plan（先寫的 RED 測試）
- 測試檔: `src/audit/redact.test.ts`（新增）
- **canary 規則（INDEX §2.1.4）**：secret 形狀的 canary **在記憶體以字串串接構造**（如 `"sk-" + "x".repeat(32)`），**不**以完整字面值寫進原始碼，避免 `scan_secrets.sh` 對 src 誤報。
- RED 測試清單:
  - [ ] secret-shaped value 在**非 secret key**（如 `resource` / 自由欄位）→ 經 `redactSecrets` 後**不含**該值、含 `[REDACTED]`。
  - [ ] 巢狀 / 陣列中的 secret-shaped value 同樣被 scrub。
  - [ ] 既有 by-key 行為不回歸（secret-named key 仍整值 `[REDACTED]`；非 secret 文字不被動）。
  - [ ] 經 `canonicalizeAuditEvent`（resource 帶 secret 形狀）→ canonical bytes **不含** canary（與 S0.2 串接）。
- 首次紅燈證據（貼 exit≠0；value-scanning 尚未實作 → canary 存活）:
  ```

  ... FAIL  expected output NOT to contain the canary
  exit code: 1
  ```
